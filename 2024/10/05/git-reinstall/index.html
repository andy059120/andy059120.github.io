Git & HomeBrew 突然都不見的悲慘故事

事情是這樣的，昨天我心血來潮把 macOS 做更新，算是有更主板號的大更新（Sonoma => Sequoia）。
然後今天用 VSCode 打開專案時，VSCode 顯示：找不到 Git。安裝它或使用 "git.path" 設置。
然後我去打指令 where git：git not found
接著去打指令 where brew：brew not found
我：……

修復嘗試一

我先回去檢查 zshrc，剛好我前幾天裝了 Oh My Zsh，行數變多的檔案看得我有點眼花，但最後看路徑還在。

修復嘗試二

之後去網路上爬文，看到兩篇文章，都說是把 Xcode 重新安裝就好，我試過之後還是沒好。（參考文章放在下面）

修復嘗試三

後來重新安裝 HomeBrew，用 HomeBrew 把 Git 順利載回來了，VSCode 設定一下之後看起來也是順利運作了，歡喜收場。

後記

原本的 Git 看起來我是用 Command_Line_Tools_for_Xcode 那邊的，現在轉成 brew，所以相關的環境設定也要一併調整，後續也處理了蠻久。
當時更新完之後其實不只是 Git & HomeBrew 不能用，我連 Python, JavaScript Node.js 都執行不起來，差點以為世界要毀滅了XD

我電腦的型號

MacBook Air (M2, 2022)
